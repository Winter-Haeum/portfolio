# Portfolio

프론트엔드 포트폴리오 프로필 페이지입니다.  
단순한 정보 나열이 아닌, **정보 흐름·인터랙션·반응형 구조 설계**에 집중해 제작했습니다.

---

## 🎯 프로젝트 목표

1. **정보 흐름이 한눈에 보이도록 구성**
2. **인터랙션은 작게, 하지만 분명하게**
3. **반응형은 구간별로 재구성**

---

## 🧩 디자인 & 기획 의도

이 포트폴리오는  
면접관이나 협업자가 처음 마주하는 **저의 첫인상**이라는 생각으로 설계했습니다.

- 밝은 톤의 컬러 블록과 굵은 테두리를 활용해  
  각 섹션이 명확히 구분되도록 디자인
- 정적인 사진 대신 캐릭터 이미지를 사용해  
  친근하고 기억에 남는 인상 표현
- 짧은 시간 안에  
  **누구인지 → 어떤 태도를 가진 사람인지 → 무엇을 할 수 있는지**  
  자연스럽게 파악할 수 있는 구조를 목표로 함

---

## 🗂 화면 구조 & 정보 흐름

### Header
- 이름 / 직군 / 캐릭터를 한 화면에 배치
- 페이지 진입 시 정체성이 바로 전달되도록 구성

### Main
- **About Me**: 가치관
- **Strengths**: 업무 태도
- **Stack Skill**: 기술 역량
- **Contact**: 바로 연결 가능한 연락 수단
- **Footer**: 마무리 멘트로 감성적인 인상 추가

---

## 📱 반응형 설계 전략

모든 해상도에서 동일한 화면을 유지하기보다는  
**해상도별로 우선순위를 다르게 설정**했습니다.

- **Desktop / Tablet (834px 이상)**  
  → 2열 그리드 구조 유지
- **600px 부근**  
  → 레이아웃 겹침 방지를 위해  
     Stack Skill / Contact 영역 1열 구조로 변경
- **Mobile (393px)**  
  → hover의 의미가 약한 환경으로 판단  
     Strengths 영역을 완전히 다른 레이아웃으로 재구성  
     (가독성과 정보 전달 우선)

---

## 🛠 적용한 주요 설정

반응형 전환 과정에서  
grid와 flex가 중첩된 구조로 인해  
자식 요소가 부모 영역을 밀어내는 문제가 발생했습니다.

이를 단순히 크기를 제한하는 방식이 아니라,  
**요소가 줄어들 수 있도록 허용하는 방향**으로 해결했습니다.

- grid 자식 요소의 최소 너비 제한 해제
- flex 내부 텍스트 영역의 줄어듦 허용
- 긴 텍스트는 줄바꿈 처리로 레이아웃 안정화

이 과정을 통해  
중첩된 grid / flex 구조에서도  
겹침 없이 안정적인 반응형 레이아웃을 구현했습니다.
