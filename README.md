# Portfolio

프론트엔드 포트폴리오 프로필 페이지입니다.  
단순한 정보 나열이 아닌, **정보 흐름·인터랙션·반응형 구조 설계**에 집중해 제작했습니다.

---

## 🎯 프로젝트 목표

1. **정보 흐름이 한눈에 보이도록 구성**
2. **인터랙션은 작게, 하지만 분명하게**
3. **반응형은 구간별로 재구성**

---

## 🧩 디자인 & 기획 의도

이 포트폴리오는  
면접관이나 협업자가 처음 마주하는 **저의 첫인상**이라는 생각으로 설계했습니다.

- 밝은 톤의 컬러 블록과 굵은 테두리를 활용해  
  각 섹션이 명확히 구분되도록 디자인
- 정적인 사진 대신 캐릭터 이미지를 사용해  
  친근하고 기억에 남는 인상 표현
- 짧은 시간 안에  
  **누구인지 → 어떤 태도를 가진 사람인지 → 무엇을 할 수 있는지**  
  자연스럽게 파악할 수 있는 구조를 목표로 함

---

## 🗂 화면 구조 & 정보 흐름

### Header
- 이름 / 직군 / 캐릭터를 한 화면에 배치
- 페이지 진입 시 정체성이 바로 전달되도록 구성

### Main
- **About Me**: 가치관
- **Strengths**: 업무 태도
- **Stack Skill**: 기술 역량
- **Contact**: 바로 연결 가능한 연락 수단
- **Footer**: 마무리 멘트로 감성적인 인상 추가

---

## 📱 반응형 설계 전략

모든 해상도에서 동일한 화면을 유지하기보다는  
**해상도별로 우선순위를 다르게 설정**했습니다.

- **Desktop / Tablet (834px 이상)**  
  → 2열 그리드 구조 유지
- **600px 부근**  
  → 레이아웃 겹침 방지를 위해  
     Stack Skill / Contact 영역 1열 구조로 변경
- **Mobile (393px)**  
  → hover의 의미가 약한 환경으로 판단  
     Strengths 영역을 완전히 다른 레이아웃으로 재구성  
     (가독성과 정보 전달 우선)

---

## 🛠 적용한 주요 설정

반응형 전환 과정에서  
그리드 안에 다시 그리드와 플렉스가 중첩된 구조로 인해  
자식 요소가 부모 영역을 밀고 나가며 레이아웃이 깨지는 문제가 발생했습니다.

이를 해결하기 위해 단순히 `width` 값을 제한하는 방식이 아니라,  
**요소가 줄어들 수 있도록 허용하는 설정**을 중심으로 조정했습니다.

```css
/* 그리드 자식이 폭을 고집하지 않도록 허용 */
.profile-info > * {
  min-width: 0;
}

/* 플렉스 내부 텍스트 영역이 부모를 밀지 않도록 설정 */
.tech-stack-content,
.contact-field {
  min-width: 0;
}

/* 그리드 내부에서 유연하게 줄어들 수 있도록 설정 */
.tech-stack,
.contact {
  max-width: none;
  min-width: 0;
}

/* 긴 텍스트 줄바꿈 처리 (태블릿 구간) */
overflow-wrap: anywhere;
이 설정들을 통해
그리드 안에 그리드와 플렉스가 중첩된 구조에서도
자식 요소가 부모 영역을 넘지 않고
안정적으로 레이아웃을 유지할 수 있었습니다.

```
